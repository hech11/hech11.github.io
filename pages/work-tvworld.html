TV-World was a college project where we had to simulate a small studio team. This game was the first project I worked on, where I managed and programmed a game to completion within a team over a substantial amount of time. We had 7 months to create any game we'd like as long as it fits a specific aesthetic, and we had a central game mechanic. In this game's case, the main mechanic was "Upgrading yourself", and the aesthetics were the 80s.

Roles & Responsibilities

Role Responsibilities
Main and only programmer

I had to program everything from shaders to AI to state machines and tools to create the game. As part of the assignment, we could not use any pre-existing assets, including code, so everything had to be made from scratch. The exception was that we could use free tools to help make the game. In this case, I decided to use Unity.


Since I was the only programmer on the project, I had to design and create scripts and tools for designers. Everything from a player controller to how we manage cutscenes and dialogue trees had to be made in a designer-friendly way. I took advantage of Unity's custom editors to help designers create the content within the game.
Games designer

We all agreed to contribute to the game's design as a team. TV-World is a platformer and a collectathon, so we would look at other games of the same genre to see what made those games fun. I was also tasked to create a game's design document that would contain the "essence" of the game and break down mechanics on how they worked.

Producer

We used tools such as HacknPlan to help assign and manage tasks that needed to be done. I would talk to my peers and hold meetings to ensure that we were on schedule and could overcome a problem. This was important as this game was being developed during the COVID pandemic, where we had to work from home. If any issues occurred, I had to make sure we had a plan to overcome them.
Github manager

In this project, we used GitHub as source control. My role as manager was to ensure developers would know how to use GitHub and follow a standard I created.

Tools used

  • Unity 2019 - HDRP Pipeline
  • RenderDoc
  • Visual Studio 2019
  • Github
  • HacknPlan

Some notes about the project

  • A save and load feature.
  • Controller support.
  • A dialogue system which allows designers to create a branching dialogue tree.
  • Used a hierarchical finite state machine(HFSM) to create the AI, scripted events and much more of the gameplay.
  • Minigames along with a online leaderboard for players to submit their highscores.
  • A shop to customise the main characters appearance.
  • Custom tools were made by me to help other artists and designer make the game.
